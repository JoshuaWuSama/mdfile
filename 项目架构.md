# 项目架构 - AI智能学习辅助系统（AIEDU Learning OS）

> 基于 OpenClaw Agent 架构的学校级 AI 智能学习操作系统

---

## 1. 项目定位

基于 OpenClaw 开发的 **学校级 AI 智能学习辅助系统**，核心能力：

- 教材知识图谱自动构建
- 学生知识掌握度追踪建模
- AI 驱动的错题归纳与学习规划
- 以学校/班级为单位的教学管理与横向对比
- 任课教师监督管理与教学分析

---

## 3. 技术栈

| 虚拟滚动 | vue-virtual-scroller | 大数据量列表（题库、错题列表） |
| 进度条 | nprogress | 路由切换进度条 |
| 图标 | @vicons/ionicons5 | 图标库 |
| 日期处理 | Day.js | 轻量日期库 |

### 3.2 后端技术栈

| 分类 | 技术 | 说明 |
|------|------|------|
| 框架 | FastAPI | 异步高性能 Web 框架 |
| ORM | Tortoise-ORM（最新版） | 异步 ORM，支持 SQLite |
| 数据验证 | Pydantic V2 | schemas 输入/输出验证 |
| 认证 | PyJWT + passlib[bcrypt] | JWT 令牌 + 密码哈希 |
| 日志 | loguru | 结构化日志 |
| 服务器 | Uvicorn | ASGI 异步服务器 |
| HTTP 客户端 | httpx | 异步调用 AI API |
| 配置管理 | python-dotenv | 环境变量管理 |
| 文件上传 | python-multipart | 教材/试卷文件上传 |
| 数据处理 | pandas + openpyxl | 成绩导入导出、学情分析 |
| PDF 解析 | PyMuPDF (fitz) | 教材 PDF 文本提取 |
| 任务调度 | APScheduler | 定时学习提醒、报告生成 |
| 测试 | pytest + pytest-asyncio | 异步单元测试 |

### 3.3 AI 能力层

| 分类 | 技术 | 说明 |
|------|------|------|
| LLM | DeepSeek API / OpenAI API | 知识点抽取、错题分析、学习建议 |
| Agent 框架 | OpenClaw | Agent Workflow 编排 |
| 向量存储 | SQLite + sqlite-vss 或 ChromaDB | 轻量级向量检索（教材 RAG） |
| Embedding | text2vec / sentence-transformers | 文本向量化 |
| 知识图谱 | NetworkX + JSON 存储 | 轻量知识图谱（SQLite 存储） |
| RAG | LlamaIndex 或 LangChain | 教材知识检索增强生成 |

### 3.4 数据库

| 类型 | 技术 | 说明 |
|------|------|------|
| 主数据库 | SQLite | 轻量部署，Tortoise-ORM 驱动 |
| 向量检索 | ChromaDB / sqlite-vss | 教材内容向量检索 |
| 缓存（可选） | 内存缓存 / diskcache | 热数据缓存 |

## 5. 数据库设计

### 5.1 ER 关系总览

```
School ──1:N── Class ──1:N── Student
  │                │
  └──1:N── Teacher │
                   ├──1:N── Homework ──1:N── HomeworkQuestion ──N:1── Question
                   │                                                     │
                   └──1:N── Exam ──1:N── ExamQuestion ──N:1── Question   │
                                                                         │
Subject ──1:N── KnowledgeNode（自关联树）                                │
  │                    │                                                 │
  └──1:N── Textbook    └──N:M── QuestionKnowledgeMap ────────────────────┘
              │
              └──1:N── Chapter

Student ──1:N── SubmissionAnswer ──N:1── Question
    │
    ├──1:N── WrongQuestion ──N:1── Question
    │
    ├──1:N── StudentKnowledgeState ──N:1── KnowledgeNode
    │
    ├──1:N── ExamResult ──N:1── Exam
    │
    └──1:N── LearningPlan
```

### 5.2 核心表结构

#### 用户系统

```sql
-- 用户表
user
├── id              INTEGER PRIMARY KEY
├── username        VARCHAR(50) UNIQUE NOT NULL
├── password_hash   VARCHAR(255) NOT NULL
├── real_name       VARCHAR(50)
├── role            VARCHAR(20) NOT NULL  -- admin / school_admin / teacher / student
├── school_id       INTEGER FK → school.id
├── is_active       BOOLEAN DEFAULT TRUE
├── created_at      DATETIME
└── updated_at      DATETIME
```

#### 学校组织

```sql
-- 学校表
school
├── id              INTEGER PRIMARY KEY
├── name            VARCHAR(100) NOT NULL
├── region          VARCHAR(100)
├── address         VARCHAR(255)
├── contact_phone   VARCHAR(20)
├── created_at      DATETIME
└── updated_at      DATETIME

-- 班级表
class
├── id              INTEGER PRIMARY KEY
├── school_id       INTEGER FK → school.id
├── grade           VARCHAR(20) NOT NULL    -- 年级：七年级、八年级...
├── name            VARCHAR(50) NOT NULL    -- 班级名：1班、2班...
├── academic_year   VARCHAR(20)             -- 学年：2025-2026
├── head_teacher_id INTEGER FK → teacher.id
├── created_at      DATETIME
└── updated_at      DATETIME

-- 学生表
student
├── id              INTEGER PRIMARY KEY
```

## 6. AI Agent 设计

### 6.1 Agent 工作流

```
教材上传 → 教材解析Agent → 知识图谱构建
                                ↓
题目录入 → 题目解析Agent → 题目-知识点绑定
                                ↓
学生做题(作业/考试) → 学情分析Agent → 更新学生掌握度模型
                                ↓
                     错题归纳Agent → 错题分类 + AI错因分析
                                ↓
                     学习规划Agent → 生成个性化学习计划
                                ↓
                     教师报告Agent → 班级学情报告 + 教学建议
```

### 6.2 各 Agent 职责

| Agent | 输入 | 输出 | 触发时机 |
|-------|------|------|----------|
| 教材解析Agent | 教材 PDF | 章节结构 + 知识点列表 | 教材上传时 |
| 题目解析Agent | 题目内容 | 知识点绑定 + 难度评估 | 题目创建时 |
| 学情分析Agent | 学生答题记录 | 知识掌握度更新 | 作业/考试批改后 |
| 错题归纳Agent | 错误答案 | 错因分类 + 知识薄弱点 | 出现错题时 |
| 学习规划Agent | 掌握度模型 | 周学习计划 + 推荐练习 | 每周自动 / 手动触发 |
| 教师报告Agent | 班级学情数据 | 班级分析报告 + 教学建议 | 教师查看时 / 定时生成 |
| 习题生成Agent | 知识点 + 难度要求 | 自动生成题目 | 教师触发 |

### 6.3 知识掌握度算法

采用 **简化 BKT（贝叶斯知识追踪）** 模型：

```
mastery_new = mastery_old × (1 - slip) / 
              (mastery_old × (1 - slip) + (1 - mastery_old) × guess)   -- 答对时

mastery_new = mastery_old × slip / 
              (mastery_old × slip + (1 - mastery_old) × (1 - guess))   -- 答错时

参数：
  slip  = 0.1   （会但答错的概率）
  guess = 0.25  （不会但猜对的概率，选择题）
```

工程简化版本：

```python
mastery = correct_count / attempt_count × difficulty_weight × time_decay
```
